Platform Architecture

Learning Objectives

  • Understand hardware, firmware, OS, and application layers
  • Recognize differences between mobile and desktop platforms
  • Learn about cloud and virtualization security
  • Understand Trusted Platform Module (TPM) and UEFI
  • Identify virtualization risks and best practices

Content Overview

This lesson covers the architecture of computing platforms and their security considerations. Understanding platform layers and components is essential for implementing comprehensive security measures.

Platform Architecture Layers

  1. Hardware Layer: Physical components including CPU, memory, storage, and peripherals
  2. Firmware Layer: Low-level software that controls hardware initialization and basic operations
  3. Operating System Layer: Software that manages hardware resources and provides user interfaces
  4. Application Layer: Software applications that run on top of the operating system
  5. Network Layer: Communication protocols and network interfaces
  6. Data Layer: Storage, databases, and data management systems

Hardware Security

Hardware security focuses on protecting physical computing components:

  • Physical Security: Locks, cages, biometric access controls
  • Tamper Detection: Seals, sensors, and intrusion detection
  • Hardware Encryption: Built-in encryption capabilities
  • Secure Boot: Verifies software integrity during startup
  • Hardware Security Modules (HSM): Dedicated crypto processors

Firmware and Boot Security

  • UEFI (Unified Extensible Firmware Interface): Modern replacement for BIOS
  • Secure Boot: Prevents unauthorized OS loading
  • Measured Boot: Creates chain of trust during boot process
  • TPM (Trusted Platform Module): Hardware-based security functions

Trusted Platform Module (TPM)

TPM is a hardware-based security feature that provides:

  • Cryptographic Functions: Key generation and management
  • Platform Integrity: Measurement and attestation
  • Secure Storage: Protected key storage
  • Remote Attestation: Platform state verification

Operating System Security

  • User Account Control: Privilege management
  • File System Permissions: Access control lists
  • Security Policies: Group policies and security settings
  • Patch Management: Regular security updates
  • Logging and Auditing: Security event monitoring

Mobile vs. Desktop Platform Security

Aspect Mobile Platforms Desktop Platforms
Physical Security Device locks, biometrics Physical access controls
App Security App sandboxing, permissions Software installation controls
Network Security Mobile-specific protocols Enterprise network security
Data Protection Device encryption, remote wipe Full disk encryption

Virtualization Security

Virtualization introduces additional security considerations:

  • Hypervisor Security: Protecting the virtualization layer
  • VM Isolation: Preventing cross-VM attacks
  • Resource Management: Controlling resource allocation
  • Snapshot Security: Protecting VM snapshots
  • Live Migration Security: Secure VM movement between hosts

Virtualization Risks

  • VM Escape: Breaking out of virtual machine isolation
  • Hypervisor Vulnerabilities: Exploiting the virtualization software
  • Side-Channel Attacks: Extracting information from shared resources
  • Configuration Errors: Misconfigured virtual networks or storage
  • Resource Exhaustion: DoS attacks on shared resources

Cloud Platform Security

Cloud security requires understanding shared responsibility models:

  • Infrastructure as a Service (IaaS): Provider secures infrastructure, customer secures applications
  • Platform as a Service (PaaS): Provider secures platform, customer secures applications and data
  • Software as a Service (SaaS): Provider secures everything, customer manages user access

Cloud Security Best Practices

  • Use encryption for data at rest and in transit
  • Implement proper access controls and identity management
  • Regular security assessments and penetration testing
  • Monitor and log all activities
  • Implement backup and disaster recovery plans
  • Understand and comply with data residency requirements

Platform Attack Surface

The platform attack surface includes all points where an attacker can try to enter or extract data from the platform. This includes hardware interfaces, firmware, operating systems, applications, and network connections.

UEFI

Unified Extensible Firmware Interface (UEFI) is a modern firmware interface for computers, replacing BIOS. It provides faster boot times, enhanced security features like Secure Boot, and a flexible pre-boot environment.

Chain of Trust

The chain of trust is a security concept where each component in the boot process verifies the integrity of the next component before execution. This ensures that only trusted software is loaded during startup.

TPM Versions and Limitations

Trusted Platform Module (TPM) provides hardware-based security functions such as secure key storage and platform integrity measurements. TPM 1.2 and TPM 2.0 differ in capabilities, with TPM 2.0 offering enhanced algorithms and flexibility. Limitations include hardware compatibility and potential vulnerabilities in implementation.

Virtualization Types

Virtualization allows multiple operating systems to run on a single physical machine by abstracting hardware resources. Types include full virtualization, para-virtualization, and hardware-assisted virtualization.

Hypervisor Types

Hypervisors manage virtual machines and can be:

  • Type 1 (Bare-metal): Runs directly on hardware, offering better performance and security.
  • Type 2 (Hosted): Runs on top of a host operating system, easier to set up but less secure.

Risks: Hyperjacking and VM Escape

Hyperjacking involves compromising the hypervisor to gain control over all virtual machines. VM escape allows a virtual machine to break out of its isolated environment and interact with the host or other VMs, posing serious security risks.

Misconfigurations

Misconfigurations in virtualization environments, such as improper access controls or network settings, can lead to vulnerabilities and unauthorized access.

Isolation Techniques

Isolation techniques help contain threats within virtual environments by separating VMs, using sandboxing, and enforcing strict access controls.

Networking

Virtual networks connect virtual machines and can be configured with firewalls, VLANs, and segmentation to enhance security.

Snapshots Security

Snapshots capture the state of a VM at a point in time but can pose security risks if sensitive data is stored or if snapshots are not properly managed and deleted.

Best Practices

Best practices for platform security include regular updates, continuous monitoring, strict access controls, secure configuration, and incident response planning.
